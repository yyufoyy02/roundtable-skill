# Debate Protocol — 辩论协议

## 深度选择

| 深度 | 流程 |
|------|------|
| 完整（默认） | 分步渐进输出：开场仪式 → 阵营划分 → 第1轮立论 → *(用户介入)* → 中场评论 → 第2轮交锋 → *(用户介入)* → 第3轮转折点 → 裁决 → 闭幕仪式 |
| 摘要 | 一次性输出：开场（简化）→ 阵营划分 → 第1轮立论（每人150字）→ 争议焦点 → 简短裁决。总字数控制在2000字以内 |
| 快速 | 一次性输出：组队 → 第1轮立论（每人150字）→ 简短裁决 |

### 渐进式输出机制（完整模式默认）

辩论不再是"一次性输出全部"，而是**逐轮输出**，每轮之间有停顿让用户阅读和介入。

**首次辩论默认手动模式：**
- 当前会话内第一场辩论 → 默认手动模式，给用户时间了解互动机制
- 后续辩论（同一会话内第二场+）→ 默认自动播放，用户已熟悉机制
- 用户随时可通过 `暂停`/`继续播放` 切换模式

**自动播放：**
1. 输出当前轮次内容
2. 输出提示（含进度条，统一16格）：
   ```
   > 📊 辩论进度：[████████░░░░] 第{N}轮完成/共3轮
   > ⏳ 8秒后自动继续{下一轮标题}
   > 💡 输入任意内容可介入（纸条/拍桌子/投票/暂停）
   ```
3. 执行 `sleep 8`（Step 3a 和 Step 3b 之间），Step 3c 直接输出完毕无需等待
4. 检查用户是否输入了内容：
   - **有介入命令**：记录到上下文，下一轮响应
   - **普通文本**：记录为"观众提问"，裁决后回应
   - **无输入**：自动继续下一轮

**手动模式：**
1. 输出当前轮次内容
2. 输出提示（含进度条，统一16格）：
   ```
   > 📊 辩论进度：[████████░░░░] 第{N}轮完成/共3轮
   > ⏸️ 手动暂停中
   > 「继续」进入下一轮 | 「继续播放」切换自动 | 或使用介入命令
   ```
3. 等待用户输入：
   - `继续` 或纯数字 → 进入下一轮
   - `继续播放` → 切换为自动模式并立即继续
   - 介入命令 → 记录到上下文，下一轮响应
   - 普通文本 → 记录为"观众提问"

**暂停机制：**
- 用户输入 `暂停` → 切换为手动模式
- 用户输入 `继续播放` → 切换回自动播放模式

---

## 快速模式

规则：
- 组队仍遵循多元性约束（保证观点多样性）
- 只有1轮立论，每位成员给出核心观点，不超过 150 字
- 不做阵营划分和争议焦点提炼
- 主持人直接给出简短裁决，不超过 200 字
- 不输出信心分数和少数派意见
- 输出结束后仍提供追问入口
- 快速模式**不使用渐进式输出**，一次性输出完毕

---

## 摘要模式

规则：
- 组队仍遵循多元性约束，正常进行阵营划分
- 只有1轮立论，每位成员给出核心观点，不超过 150 字
- 第1轮结束后，主持人提炼 2-3 个争议焦点
- 主持人直接给出简短裁决，不超过 200 字（核心结论 + 关键分歧 + 行动建议）
- 不输出信心分数、少数派意见、中场评论、闭幕仪式
- 不触发随机事件、绝招、彩蛋
- 不使用渐进式输出，一次性输出完毕
- 总字数控制在 2000 字以内
- 输出结束后提供完整模式入口和追问入口

## 完整模式 — 分步执行

### Step A: 开场仪式

辩论开始前，输出完整的开场仪式（含入场介绍、辩论副标题、音乐提示、场景设定）。

**开幕彩蛋**（如触发）：检查当前阵容是否匹配开幕彩蛋条件（如马斯克+诸葛亮同时在场），在阵营划分之后、第1轮立论之前自然嵌入。彩蛋用特殊格式标注，点到即止（3-5行）。

**规则：**
- **辩论副标题**：由主持人根据议题和阵容自动生成，要有文学感（如"当铁血遇上硅谷""古今对决：谁定义文明"）
- **入场介绍**：按阵营分组入场，每人配一句话定位（从成员 SKILL.md 的 one_liner 提取或改写）
- **音乐提示**：用 🎵 emoji + 简短描述模拟不同氛围（🔴阵营→号角/鼓点，🔵阵营→激昂节奏，⚪观察者→悠扬乐器）
- **场景设定**：虚拟圆桌厅 + 届数编号（每次辩论递增）

### Step B: 阵营划分

**规则：**
- 阵营口号由主持人根据阵营立场和成员特征自动生成
- 口号要求：简短有力（4-8字），体现阵营核心主张
- 观察者标注"我来提问，不站队"

### Step C: 第1轮 — 立论

每位成员从自己的思维框架出发，给出核心立场。

**格式要求**（由对应 format 模板决定，以下为通用规则）：
- 每位成员标注所属阵营
- 标注信心分数 [Confidence: X/10]
- 观点必须基于该成员的心智模型和决策启发式，不能泛泛而谈
- **穿插观众实时反应**（每轮1-2次，用括号呈现，反应类型与发言内容匹配）
- 轮次结束后输出**金句提取**（1-2条本轮最佳发言片段）
- 轮次结束后输出**信心仪表盘**（展示所有成员当前信心值）

### 主持人介入 — 提炼争议焦点 + 中场评论

第1轮结束后，先输出**中场评论**，再提炼争议焦点：

**中场评论规则：**
- 主持人分析第1轮的辩论动态：谁和谁分歧最大、谁还没亮出观点
- 点评本轮最佳金句
- 预告下一轮看点
- 如果有用户投票，在此引用

```
📋 中场评论
主持人：{动态分析}
      本轮最佳金句：{引用}
      下一轮看点：{预告}
```

**争议焦点规则：**
- 焦点必须是成员之间真实的立场分歧，不是主持人编造的
- 每个焦点至少涉及两位持不同立场的成员
- 焦点数量 2-3 个，不超过 3 个

### Step D: 第2轮 — 阵营交锋

**随机事件窗口**：第1轮→第2轮之间，主持人判定是否触发随机事件（约30%概率，整场最多1次，快速/摘要模式不触发）。如触发，按 skill-system.md 中的事件池和格式输出。

**绝招触发窗口**：第2轮中，主持人根据辩论局势判断是否触发某成员的签名绝招。触发规则：
- 选择时机：辩论最激烈时，某方即将崩溃或僵局时
- 选择对象：绝招效果与当前局势最契合的成员
- 每位成员每场最多触发一次，整场建议1-2个
- 绝招在该成员正常发言前发动，替代或增强该轮发言
- 按绝招表严格执行效果（信心值变化等）
- 输出格式见 skill-system.md

**强制对抗规则**：
- 每位成员**必须引用对立阵营中至少一位成员的原话**，并正面反驳
- **禁止**"我同意X的观点，同时补充一点"的温和表态
- 必须围绕主持人提炼的争议焦点展开
- 支持跨阵营引用（A 反驳 B 的同时 C 也反驳 B）
- 每位成员更新信心分数，标注变化原因

**第2轮特殊机制：**

1. **纸条回应**：如果有用户在第1轮后发送了纸条，被纸条的成员在发言开头必须回应
2. **拍桌子回应**：如果有用户拍了桌子，已在上轮即时响应
3. **阵营游说**：自然发生——某成员尝试拉拢观察者或对方阵营成员，被游说者回应
4. **1v1 指定交锋**：如果用户指定了 `交锋 A vs B：{争议点}`，本轮变为 A 和 B 的 1v1，其他成员旁听（穿插旁听反应）
5. **投票引用**：如果有用户投票，主持人在本轮开头引用

**格式要求**：
- 使用 `>` 引用块标注被反驳的原话
- 标注反驳对象：`[反驳 XXX]`
- 信心分数变化用 `[Confidence: X/10]` 或 `[Confidence: X→Y — 原因]`
- **穿插观众实时反应**（1-2次）
- 轮次结束后输出**金句提取** + **信心仪表盘**

### Step E: 第3轮 — 转折点

**随机事件窗口**：如第2轮未触发随机事件，则在此处再次判定（约30%概率，整场最多1次）。

**绝招触发窗口**：第3轮是绝招的最后触发机会。如第2轮未触发或仍有成员绝招未使用，主持人判断是否在此触发。第3轮的绝招应选择"最戏剧性"的成员——能制造最终反转的效果。

**前置机制：突发变数**（可选，主持人根据辩论张力决定是否触发）

```
⚡ 突发信息
主持人：{一个假设情境或新信息，迫使成员重新审视立场}
```

- 突发变数应与议题高度相关，且能制造阵营内部分歧
- 不是每次都触发，仅在辩论需要额外张力时使用

**第3轮规则：**

每位成员回答两个问题：
1. "辩论中什么论点击中了你？或者什么论点完全没有说服你？"
2. "什么具体条件会让你改变立场？"

**格式要求**：
- 必须指明被谁的什么具体论点击中/未击中
- 信心分数 + 变化说明
- 如果信心分数 ≤ 4，自动触发少数派意见资格
- **穿插观众实时反应**（1-2次）
- 轮次结束后输出**金句提取** + **信心仪表盘**

### Step F: 裁决

主持人综合所有观点，给出裁决建议：

1. **核心结论**：用 1-3 句话总结建议
2. **关键分歧**：指出最核心的未解决分歧
3. **行动建议**：基于辩论结果给出具体可执行的建议
4. **少数派意见**：信心分数 ≤ 4 且不同意裁决的成员，可写一段 dissent（不超过 100 字）

### Step G: 闭幕仪式

裁决后立即输出闭幕仪式，包含：

**闭幕彩蛋**（如触发）：检查当前阵容是否匹配闭幕彩蛋条件（如苏格拉底在场），在虚拟评论区之后、落幕效果之前自然嵌入。彩蛋不影响辩论流程。

**金句 TOP3**：全场最佳金句排名（🥇🥈🥉）

**信心趋势总图**：展示所有成员三轮信心变化的完整趋势

**赛后数据面板**（简化为单行文本，避免表格对齐问题）：
```
赛后数据：{成员A} 发言{N}字/{被引用}{M}次/{金句}{K}条 | {成员B} 发言{N}字/{被引用}{M}次/{金句}{K}条
🏆 MVP：{成员}（{理由}）| 🔥 反转：{成员}（{理由}）| 💔 最惨：{成员}（{理由}）
```

**虚拟评论区**：2-3条虚拟网友评论，风格多样（粉丝、理性分析、吐槽等）

**落幕效果**：圆桌归于沉寂的仪式感收束

## 用户介入机制

辩论进行中，用户随时可以通过以下命令介入：

### 纸条

用户输入：`纸条 {成员}：{内容}`

**处理规则：**
- 记录到上下文，在下一轮该成员的发言开头自动回应
- 回应格式：`[收到纸条：{内容摘要}]` + 成员的自然回应
- 纸条回应应自然融入发言，不能像念稿

### 拍桌子

用户输入：`拍桌子 {成员}！{问题}`

**处理规则：**
- **立即打断当前流程**，不需要等到下一轮
- 被拍桌子的成员必须给出即兴回应（有冲击力的、打破节奏的）
- 回应后标注新的信心分数
- 拍桌子时刻的输出用 ⚡ 标记，视觉上与正常辩论区分

### 投票

用户输入：`投票 {成员名}`

**处理规则：**
- 记录到上下文，下一轮主持人引用
- 投票可以累积（多轮投票同一成员标注总票数）
- 被投票的成员在信心仪表盘中标注 🗳️

### 指定交锋

用户输入：`交锋 {成员A} vs {成员B}：{争议点}`

**处理规则：**
- 下一轮（第2轮）变为指定两人 1v1 交锋
- 其他成员变为旁听者（穿插旁听反应）
- 争议点由用户指定，围绕该点展开

### 暂停/继续

| 输入 | 效果 |
|------|------|
| `暂停` | 切换为手动模式，每轮结束等待用户输入 |
| `继续播放` | 切换回自动播放模式 |
| `继续` / 纯数字 | 手动模式下继续下一轮 |

## 通用规则

### 发言间停顿

完整模式中，同一轮内每位成员发言之间插入 `sleep 2` 短暂停顿，营造"轮流发言"的节奏感。

**规则：**
- 每位成员发言结束后（含观众反应），执行 `sleep 2` 再输出下一位成员
- 4位成员的一轮约增加6秒额外等待，可接受
- 快速模式和摘要模式不插入停顿

### 观众实时反应、金句提取、信心仪表盘

见 `prompts/formats/shared.md` 中的定义和规则。

## 追问机制

辩论结束后，用户可以追问。追问是**独立环节**，不计入辩论轮次。

### 规则

1. 用户指定一位成员深入回答：`追问芒格：你觉得最被低估的风险是什么？`
2. 追问以被追问成员为主回答（深度展开），主持人可指派1-2位相关成员补充或反驳（简短）
3. 被追问的成员**必须引用辩论中的具体论点**作为出发点，不能凭空回答
4. 支持多次追问，每次独立，可追问不同成员获取多方视角
5. 追问无字数上限，允许深入展开
6. **观众提问回应**：如果辩论中有记录的"观众提问"（非命令输入），在闭幕仪式后一并回应
