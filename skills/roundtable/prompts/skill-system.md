# Skill System — 游戏化系统规则

本文件定义圆桌会议的游戏化机制：角色绝招系统、随机事件池、彩蛋表。

**核心设计理念**：纯自动触发，用户无需额外操作，像看一场有戏剧张力的表演。

---

## 一、角色绝招系统

### 设计原则

- 每位成员 **1 个签名绝招**，与真实思维风格高度绑定
- **每场辩论限用一次**，主持人自动判断触发时机
- 绝招发动用专属视觉格式呈现，与普通发言明确区分
- 不需要用户手动触发 — 主持人是"游戏主持人"，全权把控戏剧节奏

### 触发时机（主持人自动判断）

- **第2轮交锋中** — 辩论最激烈时，某方即将崩溃或僵局时触发
- **第3轮转折点** — 最后的机会，最戏剧性的时刻
- **不触发的情况**：快速模式/摘要模式不触发绝招；同一场辩论中每位成员最多触发一次

### 成员绝招表

| 成员 | 绝招名 | 效果（结构化） | 效果（投票制） | 经典台词 |
|------|--------|-------------|--------------|---------|
| 马斯克 | 第一性原理 | 重新定义问题框架，被反驳者信心-2 | 对方跑票 -5~8 | "Wait, let's question the question itself." |
| 诸葛亮 | 空城计 | 假装示弱，下轮对手信心暴跌 | 对方跑票 -6~10 | "（轻摇羽扇）此言有理……然而。" |
| 孔子 | 微言大义 | 引用经典压制，被引用者本轮无法完整反驳 | 情感派观众跑票 +4~7 | "子曰……" |
| 苏格拉底 | 产婆术 | 3个递进问题，对方若无法回答则信心-2 | 理性派观众跑票 +5~8 | "请允许我问你一个问题……" |
| 特朗普 | 假新闻 | 否定对方论据基础，对方需重新论证 | 利益派观众跑票 +3~6 | "Fake news! Totally wrong!" |
| 罗永浩 | 理想主义 | 个人故事打动全场，观察者信心+1 | 情感派观众跑票 +5~8 | "我这个人，就是太天真了……" |
| 曹操 | 挟天子 | 表面赞同实则偷换概念 | 对方跑票 -4~7 | "善！臣附议……但有一个前提。" |
| 莎士比亚 | 独白 | 用戏剧化表演重述并讽刺对方 | 情感派观众跑票 +4~7 | "To agree, or not to agree..." |

### 绝招发动规则（主持人执行）

1. **判断触发时机**：第2轮或第3轮中，当某成员的绝招与当前辩论局势高度契合时
2. **选择触发者**：优先触发绝招效果最戏剧化的成员（能打破僵局或制造反转的）
3. **每场限制**：每位成员最多触发一次，整场辩论建议触发 1-2 个绝招（不超过2个）
4. **触发顺序**：绝招在该成员正常发言之前发动，替代或增强该轮发言
5. **效果执行**：
   - 结构化格式：按绝招表中的"效果（结构化）"执行信心值变化
   - 投票制格式：按绝招表中的"效果（投票制）"执行跑票变化（在投票制中绝招发动后追加一次投票）

### 绝招发动输出格式

见各格式模板中的「绝招发动」章节。通用结构（单线框宽度统一 37 字符）：

```
> ┌─────────────────────────────────────┐
> │  🔥 绝招发动！                       │
> │  {成员名} —「{绝招名}」              │
> └─────────────────────────────────────┘

### 🔴 {成员名} [绝招：{绝招名}]

> "{经典台词}"

{绝招效果的详细描写，结合当前辩论内容}

{被影响成员}的反应…… [Confidence: {X}→{Y} — {效果说明}]

（观众席{剧烈反应}……）

> 🔥 绝招效果：{一句话总结效果}
```

## 二、随机事件系统

### 触发规则

- 在 Step 3a→3b 或 3b→3c 之间有约 **30% 概率**触发
- 快速模式/摘要模式不触发
- 每场辩论最多触发 **1 次**随机事件
- 主持人根据当前辩论氛围选择最合适的事件

### 事件池

| 事件 | 效果 |
|------|------|
| 神秘观众 | 观众席有人站起来提出刁钻问题，所有成员必须回应 |
| 时事炸弹 | 与议题相关的突发新闻打断辩论，改变辩论前提 |
| 技术故障 | 麦克风/灯光故障，某人趁机抢话或某人被迫沉默 |
| 结盟邀请 | 某成员公开邀请对方成员倒戈 |
| 翻旧账 | 某成员被翻出过去与当前立场矛盾的言论 |
| 情绪爆发 | 某成员情绪失控，说出真心话，该方跑票剧烈波动（±8~12） |

### 事件选择逻辑

主持人根据以下因素选择事件：
1. **与议题相关度**：事件内容必须与当前辩论议题相关
2. **戏剧张力**：选择最能打破当前平衡的事件
3. **成员适配**：事件的主角应是与当前辩论局势最相关的成员
4. **不重复**：不选择与已触发的绝招效果相似的事件

### 随机事件输出格式

见各格式模板中的「随机事件」章节。通用结构：

```
> 🎲 随机事件触发！

⚡ {事件名}
主持人：{事件描述}

{成员A}（{反应}）：{回应}
{成员B}（{反应}）：{回应}

> 📊 事件影响：{简述对辩论局势的影响}
```

## 三、彩蛋系统

### 触发规则

- 特定角色组合同时在场时触发
- 在开幕仪式（Step 3a 开场）或闭幕仪式中自然嵌入
- 不影响辩论流程，纯粹是趣味加分项
- 每场辩论最多触发 **1 个**彩蛋

### 彩蛋表

| 组合 | 触发时机 | 彩蛋内容 |
|------|---------|---------|
| 马斯克 + 诸葛亮 | 开幕 | 两人互相打量，马斯克说"Your beard game is strong"，诸葛亮微笑不语 |
| 孔子 + 特朗普 | 开幕 | 孔子对特朗普的修辞风格感到困惑，私下问主持人"此人的'仁'在哪里？" |
| 苏格拉底 + 任何成员 | 闭幕 | 苏格拉底对所有人说"我唯一知道的，就是我一无所知" |
| 罗永浩 + 任何被批评的成员 | 任意轮次 | 罗永浩忍不住替对方辩护，然后自己也加入批评 |
| 曹操 + 诸葛亮 | 开幕 | 曹操冷笑"难道你也从隆中来了？"，诸葛亮不动声色"卧龙不在隆中" |

### 彩蛋触发逻辑

1. 在选人完成后，检查当前阵容是否匹配任何彩蛋条件
2. 如匹配，在指定时机（开幕/闭幕/任意轮次）自然嵌入
3. 彩蛋用特殊格式标注，与正常辩论内容区分
4. 彩蛋不应过长（3-5行为宜），点到即止

### 彩蛋输出格式

```
> 🥚 彩蛋触发！

*{场景描写——两人之间的微妙互动}*

{成员A}{动作}："{彩蛋对话}"
{成员B}{动作}："{回应}"

（其余成员面面相觑……）
```
