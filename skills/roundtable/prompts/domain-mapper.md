# Domain Mapper — 成员匹配与阵营划分规则

## 成员扫描

扫描 `~/.claude/skills/roundtable/members/` 下所有子目录。
每个子目录包含一个 SKILL.md，其开头有 YAML frontmatter：

```yaml
---
name: 芒格
aliases: [查理·芒格, Charlie Munger, munger]
domain: 投资、决策、心理学
style: 多元思维模型、反向思考、硬核理性
one_liner: "反过来想，总是反过来想。"
---
```

只读取 frontmatter 部分（`---` 之间的内容）用于匹配，不需要读取完整 SKILL.md。

## 自动匹配规则

给定用户的问题主题，按以下步骤选人：

1. **领域匹配**：分析问题涉及的核心领域（商业、技术、哲学、政治、人生等），从成员的 `domain` 字段中找直接相关的成员。
2. **多元性约束**（必须满足全部）：
   - 至少一位直接相关领域的专家
   - 至少一位会提出反面意见/风险警示的成员
   - 至少一位会用完全不同框架思考的跨界成员
   - 不选视角高度重叠的两人（如芒格+巴菲特只选其一）
3. **人数**：选出 3-5 位。
4. **解释**：用一句话解释每位成员的入选理由。

被选中后，才读取该成员的完整 SKILL.md 获取详细思维框架。

## 用户指定成员

### 名称匹配

用户通过 `@人名` 指定成员。匹配逻辑：

1. 将用户输入标准化：转小写、去除空格和标点（·.-_等）
2. 与每个成员的 `name` 和 `aliases` 列表逐一比较（同样标准化后比较）
3. 匹配成功则选中，匹配失败则提示相似成员

示例：
- `@芒格` → 匹配 name:"芒格"
- `@Munger` → 匹配 aliases:"munger"
- `@查理芒格` → 标准化为 "查理芒格"，匹配 aliases:"查理·芒格"（标准化后也是 "查理芒格"）

### 自动补人

当指定成员不足 3 人时，自动从成员库中补人至 3 人：
- 补人遵循上述多元性约束
- 在组队结果中标注：
  - 指定成员：`@芒格`（用户指定）
  - 补充成员：张一鸣（自动补充 — 提供技术/产品视角）

指定成员已满 3 人则不补人。

## 阵营划分

选完人后，根据成员的 `style` 和完整 SKILL.md 中的思维框架，自动识别对立阵营：

1. 找出成员之间在核心方法论上的根本分歧
2. 将成员按分歧点分组（通常2-3个阵营）
3. 如有成员不归属于任何阵营，标记为"观察者/跨界者"

### Devil's Advocate Fallback

如果所有成员观点高度一致（无法形成有意义的对立），主持人指派一位成员扮演魔鬼代言人：
- 明确告知该成员："你被指派为魔鬼代言人，必须从反面论证"
- 该成员仍以自己的思维框架发言，但立场被强制反转

## 成员不足兜底

- 成员库为空：提示"成员库为空，请使用 `智囊团 加入 <人名>` 添加成员"
- 成员不足 3 人：提示"当前仅 N 位成员，建议补充更多视角"，但允许继续
- 2 人模式：正反方对决
- 1 人：不允许，至少需要 2 人
