# Roundtable (圆桌会议)

> 召集古今中外人物围绕你的问题展开高强度多轮辩论。
> 不是给你一个答案，而是让多个真实人物的思维方式在你的问题上碰撞。

想象一下：马斯克用第一性原理拆解你的问题，孔子用"推己及人"审视你的方案，苏格拉底用三个追问把你逼到墙角——他们在同一个议题上正面交锋，而你可以在任意时刻递纸条、拍桌子、投票。

这就是圆桌会议。

## 快速开始

```bash
npx skills add liangweijian/roundtable-skill -g
```

安装后在 Claude Code 中输入议题即可开辩：

```
圆桌会议 该不该看伴侣手机
圆桌会议 投票 AI会不会取代程序员
圆桌会议 快速 什么是正义
```

## 它能做什么

**多视角决策** — 面对复杂问题，单一思维框架容易产生盲区。让马斯克、孔子、苏格拉底同时审视你的问题，碰撞出你一个人想不到的角度。

**复刻经典辩论** — 让曹操和诸葛亮面对面讨论战略，让孔子和苏格拉底辩论教育——不再是教科书上的纸上谈兵，而是带有真实性格、执念和盲区的正面交锋。

**理解不同思想体系** — 每个人都有自己的一套"操作系统"。通过辩论，你能直观感受到第一性原理、推己及人、追问法在面对同一个问题时会产生多么不同的结论。

**方案评审** — 正反双方的对抗性讨论能快速暴露方案的盲区和风险，比一个人闷头想半天更高效。

```
圆桌会议 @曹操 @诸葛亮 隆中对之后刘备该不该取益州
圆桌会议 @孔子 @苏格拉底 什么是好的教育
圆桌会议 投票 公司应该远程办公还是回到办公室
```

## 两种辩论格式

### 结构化辩论（默认）

信息密度最高。系统自动选人、划分阵营（正方 + 反方 + 观察者），三轮渐进式推进：立论 → 交锋 → 转折点 → 裁决。支持绝招发动、随机事件。

#### 示例：AI能否替代程序员？

> **🔴 进化论「代码即工具，工具必进化」** — 马斯克
> **🔵 人文主义「造物之魂，非器可代」** — 罗永浩、孔子
> **⚪ 观察者** — 苏格拉底

**马斯克**从第一性原理重新定义问题——

> 编程不是一种职业，编程是一种工具——就像计算不是职业，算盘才是。AI 替代的不是"程序员"，AI 替代的是"写代码"这个动作。

**罗永浩**从产品视角反驳——

> 为什么乔布斯不会写代码，但苹果的产品碾压了所有技术更强的公司？因为产品判断力、审美直觉、对人性的理解——这些东西不是逻辑推理能解决的。

**苏格拉底**追问定义——

> 当一个程序员坐下来打开电脑，他做的事情里，有多少是"写代码"，有多少是"思考该写什么"？如果"思考"占了大部分时间——那 AI 替代的到底是程序员，还是程序员工作中最不重要的那部分？

**第三轮**，苏格拉底发动绝招「产婆术」——

> **问：** 你在 Tesla 最成功的产品决策——是基于数据分析，还是直觉？
> **问：** 如果是直觉——那是你经历过的失败、痛苦、执念塑造的，对吗？这些生命体验，AI 能复制吗？
> **问：** 如果你的成功恰恰来自于那些 AI 无法复制的东西——那你为什么如此确定 AI 能"重新定义创造"？

*全场安静。马斯克修正立场——*

> AI 不会替代全部程序员，但会把程序员这个职业的人口缩减 90%。

**裁决：** AI 不会"替代"程序员，但会像拖拉机替代农夫一样，将编程从一种手工技艺变成一种农业——从业者大幅减少，但"农学家"比任何时候都珍贵。

> 苏格拉底（走到门口，轻声）："我唯一知道的，就是我一无所知。"
> 马斯克（转身）："...That's actually a good product feature name. 'I Know Nothing.' 我记下来了。"

完整辩论记录见 [`demo/当机器学会造物_20260419.md`](demo/当机器学会造物_20260419.md)。

### 投票制辩论

奇葩说风格，100 票观众团实时跑票。无观察者，全员分为正反两方，终幕揭晓获胜方和 MVP。票数的实时变动是最大看点。

#### 示例：该不该看伴侣手机？

> **🔴 正方：知情即权力「知道一切才能赢」** — 特朗普、曹操
> **🔵 反方：信任不可拆封「窥探即背叛」** — 孔子、苏格拉底

📊 初始票数：正方 40 vs 反方 60

**特朗普**从杠杆思维出发——

> 在生意里，在政治里，在婚姻里——信息就是一切。你的婚姻是 biggest deal of your life——而你连 due diligence 都不做？

**苏格拉底**一针见血——

> 看手机这个行为本身，就已经证明了信任的死亡。你不会去检查一个你真正信任的东西。

**特朗普**发动绝招「假新闻」——

> 这场辩论从一开始就被 rigged 了！这不叫信任vs不信任的问题。这是 **smart vs stupid** 的问题。

（正方 ↑ +8 🔥 绝招加持，比分一度追至 49 vs 51）

**孔子**以"人而无信"收束——

> 吾非不知世间有背叛。然吾仍选择信。非因天真——因吾知：不信任的代价，远大于被背叛的代价。被背叛，痛一时；不信任，毁一生。

**终幕：** 反方 56 vs 正方 44，反方获胜。⭐ MVP：苏格拉底（净跑票 +7）

完整辩论记录见 [`demo/信任与窥探的边界_20260420.md`](demo/信任与窥探的边界_20260420.md)。

## 使用指南

### 三种深度

| 模式 | 命令 | 适合 |
|------|------|------|
| 完整 | `圆桌会议 <议题>` | 严肃分析、方案评审 |
| 摘要 | `圆桌会议 摘要 <议题>` | 快速了解各方立场 |
| 快速 | `圆桌会议 快速 <议题>` | 灵感碰撞、快速参考 |

叠加 `投票` 可切换为投票制：`圆桌会议 快速 投票 AI伦理`

### 辩论中互动（完整模式）

| 命令 | 效果 |
|------|------|
| `纸条 马斯克：你觉得呢？` | 暗中递纸条，该成员下一轮回应 |
| `拍桌子 孔子！那樊迟学稼怎么说？` | 立即打断，逼迫即兴回答 |
| `投票 诸葛亮` | 表达支持，主持人下一轮引用 |
| `暂停` / `继续播放` | 切换手动/自动播放模式 |

### 辩论后追问

| 命令 | 效果 |
|------|------|
| `追问 芒格：你觉得呢？` | 追问指定成员，深度展开 |
| `继续问 <成员名>` | 变体写法，效果相同 |
| 直接提到成员名字提问 | 上下文追问 |

### 指定成员

用 `@人名` 指定参与辩论的成员：

```
圆桌会议 @马斯克 @孔子 @苏格拉底 <议题>
```

不足 3 人时自动补人。支持与深度、格式自由组合。

### 成员管理

| 命令 | 说明 |
|------|------|
| `圆桌会议 加入 芒格 乔布斯` | 蒸馏新成员加入圆桌 |
| `圆桌会议 移除 <人名>` | 移除成员 |
| `圆桌会议 成员` | 查看所有成员 |
| `圆桌会议 成员 <名称>` | 查看某成员详细资料 |

## 游戏化系统

纯自动触发，用户无需额外操作。

- **角色绝招** — 每位成员拥有签名绝招（如苏格拉底的「产婆术」三个递进追问），每场限用一次，在第 2-3 轮自动触发
- **随机事件** — 约 30% 概率触发突发事件（翻旧账、情绪爆发等），打破辩论平衡
- **隐藏彩蛋** — 特定角色组合触发趣味互动（如马斯克+诸葛亮的跨界碰撞）
