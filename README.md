# Roundtable (圆桌会议)

> 召集古今中外人物围绕你的问题展开高强度多轮辩论。
> 不是给你一个答案，而是让多个真实人物的思维方式在你的问题上碰撞。

想象一下：马斯克用第一性原理拆解你的问题，孔子用"道器之分"审视你的方案，苏格拉底用三个追问把你逼到墙角——他们在同一个议题上正面交锋，而你可以在任意时刻递纸条、拍桌子、指定对决。

这就是圆桌会议。

## 安装

```bash
npx skills add liangweijian/roundtable-skill -g
```

安装后在 Claude Code 中输入 `圆桌会议 <你的议题>` 即可开辩。

## 示例辩论

以下是真实运行记录（议题：**AI能否替代程序员？**），展示系统如何自动选人、划分阵营、触发绝招和推进辩论。

### 阵容揭晓

系统根据议题自动匹配了 4 位成员，划分为两个阵营 + 观察者：

> **🔴 进化论「代码即工具，工具必进化」** — 马斯克
> **🔵 人文主义「造物之魂，非器可代」** — 罗永浩、孔子
> **⚪ 观察者** — 苏格拉底

### 第一轮：群雄立论

马斯克从第一性原理出发，重新定义问题——

> 编程不是一种职业，编程是一种工具——就像计算不是职业，算盘才是。当计算器发明之后，你会问"计算器能否替代打算盘的人"吗？……AI 替代的不是"程序员"，AI 替代的是"写代码"这个动作。

罗永浩从产品视角反驳——

> 为什么乔布斯不会写代码，但苹果的产品碾压了所有技术更强的公司？因为产品判断力、审美直觉、对人性的理解——这些东西不是逻辑推理能解决的……编程从来不只是写代码，编程是创造产品。

孔子从教育角度切入——

> 君子不器。学编程之人，所学者非代码也，乃逻辑、秩序、创造之力。AI 能代写代码，然能代人"学"乎？AI 如良马，能日行千里。然驭马之人，需识途、知进退、明方向。

苏格拉底没有急于表态，而是追问定义——

> 当一个程序员坐下来打开电脑，他做的事情里，有多少是"写代码"，有多少是"思考该写什么"？如果"思考"占了大部分时间——那 AI 替代的到底是程序员，还是程序员工作中最不重要的那部分？

### 第二轮：绝招交锋

马斯克发动签名绝招「第一性原理」，将辩论框架升级——

> 让我从第一性原理拆解：编程的本质是什么？是把人类意图翻译成机器能执行的指令……不是"AI 替代程序员"，是 AI 在重新定义"创造"本身。而那些只愿意在旧框架里思考的人——都会被淘汰。

罗永浩被击中（信心 7→5），但他随即反击——

> 你在 Twitter 上做那些决策——裁员 80%、改算法——是靠分析用户行为数据做的吗？不是。是凭直觉、凭个人审美、凭你对"自由言论"的个人信念做的。创造从来不是效率问题。你造火箭服从物理定律，但你做产品面对的是人，人不服从第一性原理。

孔子以"弓箭之喻"回应——

> AI 亦然——AI 是箭，人是弓。箭可更利，然弯弓射箭之人，须有志向、有判断、有担当。虽小道必有可观者焉，致远恐泥，是以君子不为也。

### 第三轮：产婆术 + 转折

主持人抛出突发信息：某科技巨头 AI 编程助手通过 Senior Engineer 测试。随后苏格拉底发动绝招「产婆术」，三个递进问题将马斯克逼入死角——

> **问：** 你在 Tesla 最成功的产品决策——转型电动车、自研芯片——是基于数据分析，还是直觉？
> **问：** 如果是直觉——那是你经历过的失败、痛苦、执念塑造的，对吗？这些生命体验，AI 能复制吗？
> **问：** 如果你的成功恰恰来自于那些 AI 无法复制的东西——那你为什么如此确定 AI 能"重新定义创造"？

*全场安静。马斯克沉默了几秒，随后修正立场——*

> 这些人类独有的东西——直觉、信念、执念——它们是珍贵的，但也是低效的。99% 的编程工作不需要这些东西。AI 不会替代全部程序员，但会把程序员这个职业的人口缩减 90%。

### 裁决

> AI 不会"替代"程序员，但会像拖拉机替代农夫一样，将编程从一种手工技艺变成一种农业——从业者大幅减少，但"农学家"比任何时候都珍贵。
>
> **行动建议：** 1) 从"写代码"转向"定义问题"；2) 比拼"知道该写什么"的判断力而非写代码速度；3) 编程教育应从"语法训练"转向"思维训练"

### 闭幕彩蛋

> 苏格拉底（走到门口，轻声）："我唯一知道的，就是我一无所知。"
>
> 马斯克（转身）："...That's actually a good product feature name. 'I Know Nothing.' 我记下来了。"
>
> （全场沉默了一瞬……然后爆发出笑声）

## 核心特性

- **真实人物思维模型** — 每位成员基于深度蒸馏，拥有独立的决策框架、表达DNA和辩论弹药
- **多轮渐进式辩论** — 立论 → 交锋 → 转折点 → 裁决，四幕完整叙事
- **实时用户介入** — 递纸条、拍桌子、投票、指定交锋，随时影响辩论走向
- **两种输出格式** — 结构化（默认）、廷议（古风投票制）
- **三种深度模式** — 完整（3轮）、摘要（1轮+裁决）、快速（一次性输出）
- **智能选人** — 自动根据议题领域匹配成员，确保观点多元性和对抗性
- **角色绝招系统** — 每位成员拥有签名绝招（第一性原理、空城计、产婆术等），主持人根据辩论局势自动触发
- **随机事件 & 彩蛋** — 轮次间随机触发突发事件，特定角色组合触发隐藏彩蛋

## 辩论格式

### 结构化（默认）

标准辩论格式，信息密度最高。阵营划分、轮次推进、信心仪表盘、金句提取、赛后数据面板一应俱全。适合严肃议题分析和方案评审。

```
圆桌会议 AI会取代程序员吗
```

每轮每人 250 字，三轮完整叙事，附带中场评论和闭幕仪式。

### 廷议

古风投票制。用户为"主公"，成员以"臣"自称（现代人物保持原有风格，仅加"臣"前缀）。三轮流程：诸臣陈词 → 朝堂辩论 → 臣等表态，最终以投票表格呈现廷议结果。

```
圆桌会议 廷议 公司是否应该全员远程办公
```

每轮每人 250 字，最终输出投票结果表，以"主公，请定夺"收尾。

## 游戏化系统

纯自动触发，用户无需额外操作 — 主持人全权把控戏剧节奏。

### 角色绝招

每位成员拥有 1 个签名绝招，与真实思维风格高度绑定。每场辩论限用一次，在第 2-3 轮自动触发（快速模式不触发）。

| 成员 | 绝招 | 效果 |
|------|------|------|
| 马斯克 | 第一性原理 | 重新定义问题框架，被反驳者信心-2 |
| 诸葛亮 | 空城计 | 假装示弱，下轮对手信心暴跌 |
| 孔子 | 微言大义 | 引用经典压制，被引用者本轮无法完整反驳 |
| 苏格拉底 | 产婆术 | 3个递进问题，对方若无法回答则信心-2 |
| 特朗普 | 假新闻 | 否定对方论据基础，对方需重新论证 |
| 罗永浩 | 理想主义 | 个人故事打动全场，观察者信心+1 |
| 曹操 | 挟天子 | 表面赞同实则偷换概念 |
| 莎士比亚 | 独白 | 用戏剧化表演重述并讽刺对方 |

### 随机事件

轮次间约 30% 概率触发突发事件，每场最多 1 次。包含神秘观众、时事炸弹、技术故障、结盟邀请、翻旧账、情绪爆发 6 种事件。

### 隐藏彩蛋

特定角色组合同时在场时自动触发，不影响辩论流程，纯粹趣味加分项。

| 组合 | 彩蛋 |
|------|------|
| 马斯克 + 诸葛亮 | 马斯克说"Your beard game is strong"，诸葛亮微笑不语 |
| 孔子 + 特朗普 | 孔子私下问主持人"此人的'仁'在哪里？" |
| 苏格拉底 + 任何成员 | 闭幕时说"我唯一知道的，就是我一无所知" |
| 罗永浩 + 被批评者 | 忍不住替对方辩护，然后自己也加入批评 |
| 曹操 + 诸葛亮 | 曹操冷笑"难道你也从隆中来了？"，诸葛亮不动声色 |

## 命令一览

| 命令 | 说明 |
|------|------|
| `圆桌会议` | 展示主菜单 |
| `圆桌会议 <议题>` | 发起完整辩论（结构化格式） |
| `圆桌会议 快速 <议题>` | 快速模式 |
| `圆桌会议 摘要 <议题>` | 摘要模式（立论+裁决） |
| `圆桌会议 廷议 <议题>` | 廷议格式（古风投票制） |
| `圆桌会议 @马斯克 @孔子 <议题>` | 指定成员辩论 |
| `圆桌会议 加入 <人名>` | 蒸馏新成员 |
| `圆桌会议 移除 <人名>` | 移除成员 |
| `圆桌会议 成员` | 成员管理面板 |
| `圆桌会议 成员 <名称>` | 查看某成员详细资料 |
| `圆桌会议 重试` | 重试上次失败的蒸馏 |
| `圆桌会议 自测` | 系统自检 |

以上命令可自由组合，例如 `圆桌会议 快速 廷议 @孔子 AI伦理`。

### 辩论中互动

| 命令 | 效果 |
|------|------|
| `纸条 马斯克：你真的认为AI安全吗？` | 暗中递纸条，该成员下一轮回应 |
| `拍桌子 孔子！那樊迟学稼怎么说？` | 立即打断，逼迫即兴回答 |
| `投票 诸葛亮` | 表达支持，主持人下一轮引用 |
| `暂停` / `继续播放` | 切换手动/自动播放模式 |

### 辩论后追问

| 命令 | 效果 |
|------|------|
| `追问 芒格：你觉得呢？` | 追问指定成员，获取深度展开 |
| `继续问 <成员名>` | 变体写法，效果相同 |
| 直接提到成员名字提问 | 上下文追问（如"马斯克你觉得呢？"） |

```

│   ├── debate-protocol.md      # 辩论协议（轮次、互动、裁决）
│   ├── skill-system.md         # 游戏化系统（绝招、随机事件、彩蛋）
│   ├── member-management.md    # 成员管理（加入/移除/查看）
│   ├── self-test.md            # 系统自测
│   └── formats/
│       ├── shared.md           # 共享视觉元素
│       ├── structured.md       # 结构化格式模板
│       └── court.md            # 廷议格式模板
├── members/
│   ├── {内置成员}/SKILL.md      # 默认成员（git 跟踪）
│   └── custom/

└── report/                     # 会议记录存档（git 忽略）
```
