# Roundtable (圆桌会议)

> 召集古今中外人物围绕你的问题展开高强度多轮辩论。
> 不是给你一个答案，而是让多个真实人物的思维方式在你的问题上碰撞。

想象一下：马斯克用第一性原理拆解你的问题，孔子用"推己及人"审视你的方案，苏格拉底用三个追问把你逼到墙角——他们在同一个议题上正面交锋，而你可以在任意时刻递纸条、拍桌子、拍桌子投票。

这就是圆桌会议。

## 安装

```bash
npx skills add liangweijian/roundtable-skill -g
```

安装后在 Claude Code 中输入 `圆桌会议 <你的议题>` 即可开辩。

## 两种玩法

### 玩法一：结构化辩论（默认）

标准辩论格式，信息密度最高。系统自动选人、划分阵营（正方 + 反方 + 观察者），三轮渐进式推进：立论 → 交锋 → 转折点 → 裁决。支持绝招发动、随机事件、隐藏彩蛋。

```
圆桌会议 AI能否替代程序员？
```

#### 示例：AI能否替代程序员？

系统根据议题自动匹配 4 位成员，划分为两个阵营 + 观察者：

> **🔴 进化论「代码即工具，工具必进化」** — 马斯克
> **🔵 人文主义「造物之魂，非器可代」** — 罗永浩、孔子
> **⚪ 观察者** — 苏格拉底

**第一轮立论**，马斯克从第一性原理重新定义问题——

> 编程不是一种职业，编程是一种工具——就像计算不是职业，算盘才是。当计算器发明之后，你会问"计算器能否替代打算盘的人"吗？……AI 替代的不是"程序员"，AI 替代的是"写代码"这个动作。

罗永浩从产品视角反驳——

> 为什么乔布斯不会写代码，但苹果的产品碾压了所有技术更强的公司？因为产品判断力、审美直觉、对人性的理解——这些东西不是逻辑推理能解决的。

孔子从教育角度切入——

> 君子不器。学编程之人，所学者非代码也，乃逻辑、秩序、创造之力。AI 能代写代码，然能代人"学"乎？

苏格拉底没有急于表态，而是追问定义——

> 当一个程序员坐下来打开电脑，他做的事情里，有多少是"写代码"，有多少是"思考该写什么"？如果"思考"占了大部分时间——那 AI 替代的到底是程序员，还是程序员工作中最不重要的那部分？

**第三轮转折**，苏格拉底发动绝招「产婆术」，三个递进问题将马斯克逼入死角——

> **问：** 你在 Tesla 最成功的产品决策——转型电动车、自研芯片——是基于数据分析，还是直觉？
> **问：** 如果是直觉——那是你经历过的失败、痛苦、执念塑造的，对吗？这些生命体验，AI 能复制吗？
> **问：** 如果你的成功恰恰来自于那些 AI 无法复制的东西——那你为什么如此确定 AI 能"重新定义创造"？

*全场安静。马斯克修正立场——*

> AI 不会替代全部程序员，但会把程序员这个职业的人口缩减 90%。

**裁决：** AI 不会"替代"程序员，但会像拖拉机替代农夫一样，将编程从一种手工技艺变成一种农业——从业者大幅减少，但"农学家"比任何时候都珍贵。

**闭幕彩蛋：**

> 苏格拉底（走到门口，轻声）："我唯一知道的，就是我一无所知。"
>
> 马斯克（转身）："...That's actually a good product feature name. 'I Know Nothing.' 我记下来了。"
>
> （全场沉默了一瞬……然后爆发出笑声）

完整辩论记录见 [`demo/当机器学会造物_20260419.md`](demo/当机器学会造物_20260419.md)。

### 玩法二：投票制辩论

奇葩说风格，100 票观众团实时跑票。无观察者，全员分为正反两方，每轮发言后观众投票显示跑票，终幕揭晓获胜方和 MVP。票数的实时变动是最大看点。

```
圆桌会议 投票 该不该看伴侣手机
```

#### 示例：该不该看伴侣手机？

系统自动匹配 4 位成员，全员分为正反两方：

> **🔴 正方：知情即权力「知道一切才能赢」** — 特朗普、曹操
> **🔵 反方：信任不可拆封「窥探即背叛」** — 孔子、苏格拉底

📊 初始票数：正方 40 vs 反方 60（社会共识倾向不该看）

**第一轮立论**，特朗普从杠杆思维出发——

> 在生意里，在政治里，在婚姻里——信息就是一切。我见过太多人，因为他们不看、不问、不查，最后输得一无所有。你的婚姻是 biggest deal of your life——而你连 due diligence 都不做？

孔子以"推己及人"反击——

> 己所不欲，勿施于人。汝不愿人窥汝之私，何以窥人之私？

苏格拉底一针见血——

> 看手机这个行为本身，就已经证明了信任的死亡。你不会去检查一个你真正信任的东西。

曹操以兵法论婚姻——

> 与其事后哭天抢地说"我瞎了眼"，不如事前多看一眼。慕虚名而处实祸——那"信任"二字，就是最大的虚名。

**第二轮随机事件触发**——观众翻出曹操旧账"宁教我负天下人，休教天下人负我"，质疑其双重标准。苏格拉底补刀："一位主张全知全查的人，自己却不愿意被查。"

**第三轮绝地反击**，特朗普发动绝招「假新闻」——

> 这场辩论从一开始就被 rigged 了！他们把问题从"你有权知道真相吗"偷换成"信任vs控制"。这不叫信任vs不信任的问题。这是 **smart vs stupid** 的问题。

（观众投票：正方 ↑ +8 🔥 绝招加持，比分一度追至 49 vs 51）

孔子以"人而无信"收束——

> 吾非不知世间有背叛。然吾仍选择信。非因天真——因吾知：不信任的代价，远大于被背叛的代价。被背叛，痛一时；不信任，毁一生。

**终幕揭晓：** 反方 56 票 vs 正方 44 票，反方获胜。

> ⭐ **本场 MVP：苏格拉底** — 净跑票 +7 票
> "你不会去检查一个你真正信任的东西。"

完整辩论记录见 [`demo/信任与窥探的边界_20260420.md`](demo/信任与窥探的边界_20260420.md)。

## 深度模式

三种深度可自由组合，满足不同场景：

| 模式 | 流程 | 适用场景 |
|------|------|---------|
| `圆桌会议 <议题>` | 完整（3轮+裁决+闭幕） | 严肃议题分析、方案评审 |
| `圆桌会议 摘要 <议题>` | 摘要（1轮立论+裁决） | 快速了解各方立场 |
| `圆桌会议 快速 <议题>` | 快速（一次性输出） | 灵感碰撞、快速参考 |

以上均可叠加 `投票` 关键词切换为投票制格式，例如 `圆桌会议 快速 投票 AI伦理`。

## 游戏化系统

纯自动触发，用户无需额外操作 — 主持人全权把控戏剧节奏。

### 角色绝招

每位成员拥有 1 个签名绝招，与真实思维风格高度绑定。每场辩论限用一次，在第 2-3 轮自动触发（快速/摘要模式不触发）。

| 成员 | 绝招 | 效果 |
|------|------|------|
| 马斯克 | 第一性原理 | 重新定义问题框架，对方跑票 -5~8 |
| 诸葛亮 | 空城计 | 假装示弱，对方跑票 -6~10 |
| 孔子 | 微言大义 | 引用经典压制，情感派跑票 +4~7 |
| 苏格拉底 | 产婆术 | 3个递进问题，理性派跑票 +5~8 |
| 特朗普 | 假新闻 | 否定对方论据基础，利益派跑票 +3~6 |
| 罗永浩 | 理想主义 | 个人故事打动全场，情感派跑票 +5~8 |
| 曹操 | 挟天子 | 表面赞同实则偷换概念，对方跑票 -4~7 |
| 莎士比亚 | 独白 | 戏剧化表演重述并讽刺，情感派跑票 +4~7 |

### 随机事件

轮次间约 30% 概率触发突发事件，每场最多 1 次。包含神秘观众、时事炸弹、技术故障、结盟邀请、翻旧账、情绪爆发 6 种事件。

### 隐藏彩蛋

特定角色组合同时在场时自动触发，不影响辩论流程，纯粹趣味加分项。

| 组合 | 彩蛋 |
|------|------|
| 马斯克 + 诸葛亮 | 马斯克说"Your beard game is strong"，诸葛亮微笑不语 |
| 孔子 + 特朗普 | 孔子私下问主持人"此人的'仁'在哪里？" |
| 苏格拉底 + 任何成员 | 闭幕时说"我唯一知道的，就是我一无所知" |
| 罗永浩 + 被批评者 | 忍不住替对方辩护，然后自己也加入批评 |
| 曹操 + 诸葛亮 | 曹操冷笑"难道你也从隆中来了？"，诸葛亮不动声色 |

## 辩论中互动

完整模式下，你可以在任意时刻介入辩论：

| 命令 | 效果 |
|------|------|
| `纸条 马斯克：你真的认为AI安全吗？` | 暗中递纸条，该成员下一轮回应 |
| `拍桌子 孔子！那樊迟学稼怎么说？` | 立即打断，逼迫即兴回答 |
| `投票 诸葛亮` | 表达支持，主持人下一轮引用 |
| `暂停` / `继续播放` | 切换手动/自动播放模式 |

### 辩论后追问

| 命令 | 效果 |
|------|------|
| `追问 芒格：你觉得呢？` | 追问指定成员，获取深度展开 |
| `继续问 <成员名>` | 变体写法，效果相同 |
| 直接提到成员名字提问 | 上下文追问（如"马斯克你觉得呢？"） |

### 自定义成员

通过蒸馏机制加入，存放在本地 `members/custom/` 目录，不会被 git 跟踪。

```
圆桌会议 加入 芒格 乔布斯
```

蒸馏依赖 [huashu-nuwa](https://github.com/alchaincyf/nuwa-skill)，首次使用时自动安装。

## 命令一览

| 命令 | 说明 |
|------|------|
| `圆桌会议` | 展示主菜单 |
| `圆桌会议 <议题>` | 发起完整辩论（结构化格式） |
| `圆桌会议 快速 <议题>` | 快速模式 |
| `圆桌会议 摘要 <议题>` | 摘要模式（立论+裁决） |
| `圆桌会议 投票 <议题>` | 投票制辩论（奇葩说风格） |
| `圆桌会议 @马斯克 @孔子 <议题>` | 指定成员辩论 |
| `圆桌会议 加入 <人名>` | 蒸馏新成员 |
| `圆桌会议 移除 <人名>` | 移除成员 |
| `圆桌会议 成员` | 成员管理面板 |
| `圆桌会议 成员 <名称>` | 查看某成员详细资料 |
| `圆桌会议 重试` | 重试上次失败的蒸馏 |
| `圆桌会议 自测` | 系统自检 |

以上命令可自由组合，例如 `圆桌会议 快速 投票 @孔子 AI伦理`。

## 项目结构

```
skills/roundtable/
├── SKILL.md                    # 主控文件（命令解析、流程编排）
├── prompts/
│   ├── domain-mapper.md        # 成员匹配与阵营划分规则
│   ├── debate-protocol.md      # 辩论协议（轮次、互动、裁决）
│   ├── skill-system.md         # 游戏化系统（绝招、随机事件、彩蛋）
│   ├── member-management.md    # 成员管理（加入/移除/查看）
│   ├── self-test.md            # 系统自测
│   └── formats/
│       ├── shared.md           # 共享视觉元素
│       ├── structured.md       # 结构化格式模板
│       └── vote.md             # 投票制辩论格式模板
├── members/
│   ├── {内置成员}/SKILL.md      # 默认成员（git 跟踪）
│   └── custom/
│       └── {自定义成员}/SKILL.md # 用户添加（git 忽略）
├── demo/                       # 示例辩论记录
└── report/                     # 会议记录存档（git 忽略）
```
