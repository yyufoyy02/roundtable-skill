# Roundtable (圆桌会议)

召集古今中外人物围绕你的问题展开高强度多轮辩论。

不是给你一个答案，而是让多个真实人物的思维方式在你的问题上碰撞。

## 安装

```bash
npx skills add liangweijian/roundtable-skill -g
```

## 快速开始

安装后在 Claude Code 中输入：

```
圆桌会议 AI会取代程序员吗
```

## 核心特性

- **真实人物思维模型** — 每位成员基于深度蒸馏，拥有独立的决策框架、表达DNA和辩论弹药
- **多轮渐进式辩论** — 立论 → 交锋 → 转折点 → 裁决，四幕完整叙事
- **实时用户介入** — 递纸条、拍桌子、投票、指定交锋，随时影响辩论走向
- **三种输出格式** — 结构化（默认）、对话体（小说风）、廷议（古风投票制）
- **三种深度模式** — 完整（3轮）、摘要（1轮+裁决）、快速（一次性输出）
- **智能选人** — 自动根据议题领域匹配成员，确保观点多元性和对抗性
- **角色绝招系统** — 每位成员拥有签名绝招（第一性原理、空城计、产婆术等），主持人根据辩论局势自动触发
- **随机事件 & 彩蛋** — 轮次间随机触发突发事件，特定角色组合触发隐藏彩蛋

## 辩论格式

### 结构化（默认）

标准辩论格式，信息密度最高。阵营划分、轮次推进、信心仪表盘、金句提取、赛后数据面板一应俱全。适合严肃议题分析和方案评审。

```
圆桌会议 AI会取代程序员吗
```

每轮每人 250 字，三轮完整叙事，附带中场评论和闭幕仪式。

### 对话体

小说风格，以场景描写和人物动作为主线。成员在自适应场景（书房、庭院、议事厅等）中自然对话，穿插环境描写和观众反应。

```
圆桌会议 对话体 是否应该裸辞创业
```

每轮每人 200 字 + 场景描写 50 字。古今人物混搭时场景自动适配，用户以本名出现。

### 廷议

古风投票制。用户为"主公"，成员以"臣"自称（现代人物保持原有风格，仅加"臣"前缀）。三轮流程：诸臣陈词 → 朝堂辩论 → 臣等表态，最终以投票表格呈现廷议结果。

```
圆桌会议 廷议 公司是否应该全员远程办公
```

每轮每人 250 字，最终输出投票结果表，以"主公，请定夺"收尾。

## 游戏化系统

纯自动触发，用户无需额外操作 — 主持人全权把控戏剧节奏。

### 角色绝招

每位成员拥有 1 个签名绝招，与真实思维风格高度绑定。每场辩论限用一次，在第 2-3 轮自动触发（快速模式不触发）。

| 成员 | 绝招 | 效果 |
|------|------|------|
| 马斯克 | 第一性原理 | 重新定义问题框架，被反驳者信心-2 |
| 诸葛亮 | 空城计 | 假装示弱，下轮对手信心暴跌 |
| 孔子 | 微言大义 | 引用经典压制，被引用者本轮无法完整反驳 |
| 苏格拉底 | 产婆术 | 3个递进问题，对方若无法回答则信心-2 |
| 特朗普 | 假新闻 | 否定对方论据基础，对方需重新论证 |
| 罗永浩 | 理想主义 | 个人故事打动全场，观察者信心+1 |
| 曹操 | 挟天子 | 表面赞同实则偷换概念 |
| 莎士比亚 | 独白 | 用戏剧化表演重述并讽刺对方 |

### 随机事件

轮次间约 30% 概率触发突发事件，每场最多 1 次。包含神秘观众、时事炸弹、技术故障、结盟邀请、翻旧账、情绪爆发 6 种事件。

```
> 🎲 随机事件触发！

⚡ 时事炸弹
主持人：突发新闻打断辩论——某公司刚宣布全面禁止AI工具，辩论前提瞬间动摇。

马斯克（皱眉）："这恰恰证明了我的观点——恐惧创新才是真正的危险。"
诸葛亮（轻摇羽扇）："时移世易，此乃天机。"

> 📊 事件影响：辩论焦点从"是否应该"转向"如何应对"
```

### 隐藏彩蛋

特定角色组合同时在场时自动触发，不影响辩论流程，纯粹趣味加分项。

| 组合 | 彩蛋 |
|------|------|
| 马斯克 + 诸葛亮 | 马斯克说"Your beard game is strong"，诸葛亮微笑不语 |
| 孔子 + 特朗普 | 孔子私下问主持人"此人的'仁'在哪里？" |
| 苏格拉底 + 任何成员 | 闭幕时说"我唯一知道的，就是我一无所知" |
| 罗永浩 + 被批评者 | 忍不住替对方辩护，然后自己也加入批评 |
| 曹操 + 诸葛亮 | 曹操冷笑"难道你也从隆中来了？"，诸葛亮不动声色 |

```
> 🥚 彩蛋触发！

*马斯克和诸葛亮第一次碰面，两人互相打量。*

马斯克（扬起眉毛）："Your beard game is strong."
诸葛亮（轻摇羽扇，微笑不语）。

（其余成员面面相觑……）
```

## 自定义成员

通过蒸馏机制加入任意真实人物，存放在本地 `members/custom/` 目录，不会被 git 跟踪。

```
圆桌会议 加入 芒格 乔布斯
```

蒸馏依赖 [huashu-nuwa](https://github.com/alchaincyf/nuwa-skill)，首次使用时自动安装。

## 命令一览

| 命令 | 说明 |
|------|------|
| `圆桌会议` | 展示主菜单 |
| `圆桌会议 <议题>` | 发起完整辩论 |
| `圆桌会议 快速 <议题>` | 快速模式 |
| `圆桌会议 摘要 <议题>` | 摘要模式（立论+裁决） |
| `圆桌会议 对话体 <议题>` | 对话体格式 |
| `圆桌会议 廷议 <议题>` | 廷议格式 |
| `圆桌会议 @马斯克 @孔子 <议题>` | 指定成员辩论 |
| `圆桌会议 加入 <人名>` | 蒸馏新成员 |
| `圆桌会议 移除 <人名>` | 移除成员 |
| `圆桌会议 成员` | 成员管理面板 |

以上命令可自由组合，例如 `圆桌会议 快速 对话体 @孔子 AI伦理`。

### 辩论中互动

| 命令 | 效果 |
|------|------|
| `纸条 马斯克：你真的认为AI安全吗？` | 暗中递纸条，该成员下一轮回应 |
| `拍桌子 马克思！那苏联怎么解释？` | 立即打断，逼迫即兴回答 |
| `投票 诸葛亮` | 表达支持，主持人下一轮引用 |
| `闪电` | 第2轮变为限时100字快攻 |
| `交锋 A vs B：争议点` | 第2轮变为两人1v1对决 |
| `暂停` / `继续播放` | 切换手动/自动播放模式 |

## 项目结构

```
skills/roundtable/
├── SKILL.md                    # 主控文件（命令解析、流程编排）
├── prompts/
│   ├── domain-mapper.md        # 成员匹配与阵营划分规则
│   ├── debate-protocol.md      # 辩论协议（轮次、互动、裁决）
│   ├── skill-system.md         # 游戏化系统（绝招、随机事件、彩蛋）
│   └── formats/
│       ├── structured.md       # 结构化格式模板
│       ├── dialogue.md         # 对话体格式模板
│       └── court.md            # 廷议格式模板
└── members/
    ├── {内置成员}/SKILL.md      # 默认成员（git 跟踪）
    └── custom/
        └── {自定义成员}/SKILL.md # 用户添加（git 忽略）
```
